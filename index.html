MOOS（已停止开发）

C# native aot 操作系统尝试

这应该是我最终的 “Hobby OS” 尝试。我很早之前就对操作系统感兴趣了。当时看了川和秀实的《30天自制操作系统》，但只有在你亲自尝试之后才会明白，这根本不是给零基础的人看的。只是让你跟着做，啥也不给你讲。结果最后我啥鸡巴也没做出来。
后来接触到 Cosmos 操作系统套件，于是有了用 C# 做操作系统的打算。但是当时的 Cosmos 纯粹就是垃圾。不知道现在好些了没，貌似有了 GC 了。能实现 GC 还是很了不起了。但是 Cosmos 的开发速度，哎，实在是慢得让人发指。Cosmos 已经存在了十多年，貌似是我三四岁的时候就有了。他那点进度还不如我一个月的进度。19 年那会儿 native AOT 好像还没出来。我都是 native AOT 出来几个月才知道的。
Cosmos 玩了几个月之后，我又去玩 x86_64 汇编。看了看 Cosmos 的 IL2CPU，随便写了点 x86_64 汇编之后居然真的学会了。Cosmos 玩了一年之后，我又发现了一个叫 MOSA-Project 的项目。但是我觉得他们的代码大部分都是没用的，于是我 fork 了这个项目，然后把我认为没用的代码全删了，改名为 MOSA-Core。当时招揽了不少志同道合的开发者，都是“Hobby OSers”。相比于 Cosmos，MOSA-Project 确实好了不少，至少从编译速度来说是这样的。
后来我自己写了个 IL2CPU 的替代品 CS2ASM，因为 IL2CPU 是 32 位的，我用 CS2ASM 就用了 64 位。这个工具简而言之就是把 C# 的 IL 中间代码转换为 x86_64 汇编源码，然后让 NASM 把汇编编译成二进制文件。但是这东西有个致命缺点，就是太慢了。尤其是在转换初始值很大的代码时（比如长度很大的字符串）。
后来发现了 Michal Strehovsky 的 ZeroSharp，我对 native AOT 的研究也就刚刚开始了。但是他这个是真的太“骨头”了，啥都没有。使用 “new” 创建对象就会莫名报错。后来又发现了另一个叫 “Rose OS”的项目。不过那是基于 UEFI 的。好在相比 ZeroSharp，这个 “Rose OS” 是非常全面的。对 native AOT 的 .modules 节也进行了初始化，“new” 关键字也能用了。
我后来就创建了个叫 Solution1 的项目，这就是 Visual Studio 的默认名称，懒得改。我学着 “Rose OS”的方法初始化了 .modules 节（他是用 C# 初始化的 modules 节，但是我那会儿 x86_64 汇编也挺会了，就直接用汇编写了初始化 modules 节的代码）。因为之前玩了那么久操作系统了，很多代码用以前的就行了。比如说网卡、IDE、AHCI 驱动啥的。
MOOS 这个项目我还有部分时间是在学校里开发的。那时候高一啊，真的爽的批爆。因为我打算拿这个参加比赛。给信息老师说了这个想法。她说了关于比赛的信息。我觉得就是稳拿（我们班上很多傻逼甚至电脑都不会用）。那会儿自习课就能跑去信息办公室写代码玩，真爽死。后来也拿了个省一等奖，市二等奖还是可以。

不过，“Hobby OS”注定失败。你不可能在Windows和Linux以及Mac OS盛行的时代尝试分一杯羹。除非你现在在90年代。不过尝试自制操作系统确实能让你学到很多，我至少学会了x86_64汇编、C语言，这对我使用IDA Pro、x64dbg提供了很大的帮助。
